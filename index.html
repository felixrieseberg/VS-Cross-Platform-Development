C++ App Development

  1. Bitmap Plasma
  1. Native development is without a doubt the way to deliver the best possible performance on devices.
  2. The only native language all devices share is C++. Working in C++ is kind of hard.
  3. That's where Xamarin comes in: The company makes a lot of money by providing *amazing* cross-platform tools which compile C# code into native iOS, Android and Windows apps.

.NET and Xamarin

Share app logic across platforms

  • Native apps for Windows, iOS and Android devices using .NET
  • Most productive and powerful platform for creating native applications
  • Maximize code reuse with Universal Projects & Portable Class Libraries

100% APIs exposed

  • Everything you can do in xCode or ADT you can do with Visual Studio
  1. How does Xamarin work? Two products: Xamarin.iOS and Xamarin.Android. Mono.
  2. iOS: Ahead-of-Time compilation of .NET C# code directly to native ARM assembly code.
  3. Android: Compiles .NET C# down to Intermediate Language (IL), which is then Just-in-Time (JIT) compiled to a native assembly when the app launches (aka the Android way).
  4. Xamarin does not only compile C# down to Android and iOS, but it also exposes the full API to C# (in Namespaces). The idea is not to write one app that runs everywhere, but to have one toolchain with a common language to maximize reuse. The resulting apps and their controls are native.
